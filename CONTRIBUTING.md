# Contributing to 3eus

First of all, thank you for showing interest in improving **3eus**! We welcome all contributions that keep this template lightweight and efficient.

## How to Contribute

### 1. Improvements and Features
If you have an idea to optimize the MVC core or add a useful feature:
* Open an **Issue** first to discuss the change.
* Ensure the feature aligns with the "lightweight" philosophy of the project.

### 2. Bug Reports
Found a bug in the router or the model loader?
* Open an **Issue** with a clear description and steps to reproduce.
* If you have a fix, feel free to submit a **Pull Request**.

### 3. Pull Request Process
1. **Fork** the repository and create your feature branch.
2. **Follow PSR Standards:** We aim to follow PHP coding standards (PSR-4 for autoloading, PSR-12 for styling).
3. **Keep it simple:** Avoid adding heavy external dependencies unless absolutely necessary.
4. **Document:** If you change how routing or controllers work, update the README or provide examples.
5. Submit your PR against the `main` branch.

## Coding Style Guidelines
* **Namespacing:** Ensure all new classes follow the project's namespace structure.
* **Comments:** Use DocBlocks for complex methods to explain parameters and return types.
* **Error Handling:** Use Try/Catch blocks for database operations and provide meaningful error messages.

---
*Happy coding! Let's make 3eus the best lightweight PHP starter kit together.*
